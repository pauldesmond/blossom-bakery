## P1 — Must fix before announce (visible bugs)

### 1.1 `two-tier-cakes.html` is announcing itself as Ganache Drip Cakes

The page body is correct (genuine Two Tier Cakes content), but the `<head>` and image alt-text are inherited from when the page was duplicated. Google currently sees this as a duplicate of `ganache-drip-cakes.html`.

- `<title>` reads "Ganache Drip Cakes — Blossom Bakery, Chelmsford" → should be "Two Tier Cakes — Blossom Bakery, Chelmsford"
- `<link rel="canonical">` points at `/ganache-drip-cakes.html` → should be `/two-tier-cakes.html`
- `og:url`, `og:title`, `og:description`, `og:image` all from ganache-drip-cakes
- Service schema `serviceType` is "Ganache drip cakes" → should be "Two Tier Cakes"
- 9 `<img>` tags have `alt="Ganache drip cake by Blossom Bakery, Chelmsford"` → should reference two-tier cakes

Fix: rewrite the four `<meta>` tags + `<link rel="canonical">` + `<title>` + Service schema, plus sweep the alt-text on the 9 images. ~15 lines total.

## P2 — Style & consistency (Helen would notice)

### 2.1 H1 styling is inconsistent across pages

Four distinct rhythms in use:

| Style | Pages |
|---|---|
| Title Case, no period | Speciality Cakes · Children's Cakes · Ganache Drip Cakes · Numbered Cakes · Giant Cookies · Cupcakes · Biscuits · Testimonials · Scones · About Helen · Everyday Cakes |
| sentence case + period | Tray bakes. · Wedding bakes. · Afternoon tea. |
| Single word, no period | Weddings · Cupcakes |
| Other | "Two tiered cakes" (two-tier-cakes — should match nav's "Two Tier Cakes") |

Pick one and sweep. Recommend Title Case no period (matches the nav labels). The 3 outliers (Tray bakes, Wedding bakes, Afternoon tea) would become "Tray Bakes", "Wedding Bakes", "Afternoon Tea".

### 2.2 Apostrophe drift (straight ' vs curly ’)

7 pages mix both. Worst offenders:

- `about.html` — 1 straight, 5 curly (17% straight)
- `customer-reviews.html` — 5 straight, 10 curly (33%)
- `wedding-cakes.html` — 3 straight, 5 curly (38%)
- `scones.html` — 1 straight, 1 curly (50%)
- `traybakes.html` — 10 straight, 2 curly (83%)
- `wedding-bakes.html` — 6 straight, 1 curly (86%)
- `handmade-biscuits.html` — 9 straight, 1 curly (90%)

Recommend curly (’) everywhere — matches modern editorial style + matches what the JSON-LD already uses (`’` escapes).

### 2.3 Opening hours wording

Five+ different formats in active use across pages:

- `Mon-Fri 9am-5pm · Sat 9am-2pm` (regular hyphen, full am/pm)
- `Mon–Fri 9am–5pm · Sat 9am–2pm` (en-dash, full am/pm)
- `Mon–Fri 9–5  ·  Sat 9–2` (en-dash, no am/pm, **double-space** around dots)
- `Mon–Fri 9–5  ·  Sat 9–2  ·  Replies within a day` (wedding-cakes)
- `Mon–Fri 9–5  ·  Sat 9–2  ·  48 hours notice` (afternoon-tea)

Recommend single canonical: `Mon–Fri 9am–5pm · Sat 9am–2pm` (en-dash, full am/pm, single space around the middle dot). Single sweep, ~8 pages affected.

### 2.4 Phone number format

`07939 618787` on 17 pages, `07939 618 787` on `contact.html` only. Pick one — recommend `07939 618787` (matches the bakery schema's `"telephone": "+44 7939 618787"`).

## P3 — Technical / SEO (invisible to humans but matters)

### 3.1 Missing `og:url` meta tag

6 pages have `og:title` + `og:description` but no `og:url`: wedding-bakes, handmade-biscuits, traybakes, cupcakes, afternoon-tea, contact.

Without it, Facebook / WhatsApp / iMessage fall back to whatever URL the user shared (usually fine, but a missing-canonical SEO smell). Add one `<meta property="og:url" content="https://myblossombakery.co.uk/<slug>.html"/>` per page.

### 3.2 All `<img>` tags lack `width` and `height` attributes

Every visible image on every page. This causes **Cumulative Layout Shift (CLS)** as images load — a Core Web Vitals red flag that Google penalises in mobile rankings.

Fix is mechanical but needs the actual image dimensions per file. Worth a sweep: probe `images/*.{jpg,jpeg,png,webp}` with Python PIL or `identify`, write a script that adds `width="W" height="H"` to each `<img>`. ~200 images, ~30 min scripted.

### 3.3 Inline-style sprawl

Pages with most hardcoded inline styles (font-size, color hex, background hex, padding, margin):

| Page | Inline styles |
|---|---|
| speciality-and-everyday-cakes | **51** (37 font-size, 12 color hex) |
| wedding-cakes | 21 |
| cakes (Speciality) | 17 |
| index | 16 |
| cupcakes | 11 |
| afternoon-tea | 10 |
| wedding-bakes | 9 |
| two-tier-cakes | 9 |

Mostly Helen's font-size tweaks via the editor. Should consolidate the most common values into CSS classes (e.g. `.size-display { font-size: 48px }`, `.size-h2 { font-size: 28px }`) and replace the inline values. Cuts file size + makes future sweeps possible.

### 3.4 Heading hierarchy skips (low priority)

Every page jumps from `<h2>` (last section) to `<h4>` (footer column headings). Universal pattern, semantically defensible (footer != content tree), but a strict accessibility checker would flag it. Could change the footer columns to use `<h3>` or `<div class="footer__heading">` and drop the heading tag entirely. Not urgent.

## P4 — Placeholder hygiene

### 4.1 Empty scaffold elements visible (no `:empty` CSS guard)

Pages with empty placeholders not auto-hidden:

- **cupcakes.html** — 5 empty `.cup-eyebrow`, 2 empty `.cup-section-head__title`, 2 empty `.cup-section-head__num`. Most are scaffold for sections Helen hasn't filled. Either fill or hide via CSS `:empty` rule scoped to `.cup-*`.
- **wedding-bakes.html** — 1 empty page-hero numeral, 2 empty eyebrows
- **traybakes.html** — 1 empty page-hero numeral, 1 empty eyebrow
- **afternoon-tea.html** — 1 empty page-hero numeral
- **handmade-biscuits.html** — 1 empty prose-section heading
- **wedding-cakes.html** — 2 empty eyebrows (subset of the `Let's design your .` issue in P1.3)

The global `.page-hero-v2__numeral:empty { display: none }` etc rules in `styles.css` (added 2026-05-11) cover the `page-hero-v2__*` and `eyebrow-v2` patterns. The `.cup-*` family needs the same treatment, OR fill with real content. Pick.
